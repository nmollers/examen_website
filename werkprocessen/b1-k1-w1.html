Plant werkzaamheden en bewaakt voortgang

Planning maken, taken verdelen en voortgang bewaken tijdens het ontwikkelproces.

Wat heb ik gedaan?

Voor dit werkproces heb ik een projectplanning opgesteld voor een webapplicatie. Ik heb user stories geschreven en deze geprioriteerd met de MoSCoW methode. Vervolgens heb ik een sprint planning gemaakt en een Gantt chart opgesteld om de voortgang visueel weer te geven.

Mijn rol

Product Owner / Scrum Master — verantwoordelijk voor planning en voortgangsbewaking.

Gebruikte technieken

User Stories MoSCoW Prioritering Agile / Scrum Gantt Chart Sprint Planning

Tools

Jira Trello Microsoft Project Excel

Bewijsstukken

Projectplanning downloaden User stories document
GitHub repository Scrum board link

Wat ging goed?

De planning was realistisch en haalbaar. Door MoSCoW prioritering kon ik duidelijk aangeven welke features prioriteit hadden.

Wat kon beter?

In het begin had ik te optimistische tijdsinschattingen. Ik had meer buffer moeten inbouwen.

Wat heb ik geleerd?

Een goede planning is cruciaal voor het succes van een softwareproject.

Hoe pas ik dit toe?

In toekomstige projecten wil ik meer risico's en buffers in mijn planning opnemen.

Resultaat / Eindproduct

Het resultaat is een complete projectplanning met user stories, sprint planning en een werkend Scrum board.
